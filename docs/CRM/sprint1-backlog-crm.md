# Sprint 1 – CRM

## Objetivo del Sprint
Implementar las funcionalidades básicas de gestión de clientes (registro, edición y eliminación) garantizando trazabilidad entre diseño, arquitectura y repositorio.

---

## Historias seleccionadas

- EIE-1 – Registrar Cliente
- EIE-2 – Editar Cliente
- EIE-3 – Eliminar Cliente

---

## Desglose técnico por historia

### EIE-1 – Registrar Cliente
- Diseño de formulario en Figma
- Diagrama de secuencia
- Definición de entidad Cliente
- Validaciones de datos
- Documentación y versionamiento

### EIE-2 – Editar Cliente
- Flujo de actualización
- Validación de datos existentes
- Actualización en base de datos
- Actualización de diagramas si aplica

### EIE-3 – Eliminar Cliente
- Confirmación de eliminación
- Eliminación lógica o física
- Registro en documentación
