# Retrospectiva – Sprint 1

## Start (Empezar a hacer)

- Validar criterios de aceptación antes de iniciar desarrollo.
- Actualizar diagramas inmediatamente después de cambios.
- Revisar dependencias antes de mover historias a In Progress.

## Stop (Dejar de hacer)

- Dejar tareas sin actualizar en el tablero.
- Documentar únicamente al final del Sprint.

## Continue (Continuar haciendo)

- Uso correcto de ramas feature.
- Crear Pull Requests antes de integrar a dev.
- Registrar Daily Scrum.
- Mantener trazabilidad historia → commit → PR.
